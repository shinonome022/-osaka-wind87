# index.html レビュー報告書

> 対象ファイル: `index.html`
> 参照リサーチ: `research.md`
> レビュー日: 2026-03-26

---

## 総評

全体的によく作られたシングルページサイトであり、research.mdのデザイン方針やカラーテーマ、セクション構成の多くが忠実に反映されている。ただし、重要情報の欠落・アクセシビリティ・SEO・コンテンツ面で複数の改善点が存在する。特に新歓ページとしての訴求力を高めるためには、費用感の明示・メンバー紹介・ギャラリーの欠落が致命的な問題となりうる。

## 問題点・改善点一覧（優先度順）

---

### [重要度: 高]

---

#### 1. MEMBER（メンバー紹介）セクションが完全に欠落している

- **観点**: 情報の欠落、新入生向け訴求
- **詳細**: research.mdのサイト構成案には「MEMBER - メンバー紹介」セクションが明示されているが、index.htmlには存在しない。新入生がクラブを選ぶ際、「どんな先輩がいるのか」は最大の関心事の一つ。顔の見えるチームであることが体育会系少人数クラブの最大の魅力であるにもかかわらず、人的魅力が伝わらない。
- **修正方法**: `#activities` と `#join` の間に `#members` セクションを追加。部員の学年・学部・一言コメント（プレイヤー/マネージャーの別も記載）をカード形式で並べる。写真が用意できない場合はアバターアイコン＋コメントだけでも効果的。

---

#### 2. GALLERY（フォトギャラリー）セクションが完全に欠落している

- **観点**: 情報の欠落、新入生向け訴求、デザイン
- **詳細**: research.mdの構成案に「GALLERY - フォトギャラリー」セクションが必須として挙げられているが実装されていない。ウインドサーフィンのスピード感・海の爽快感・仲間との記憶はテキストではなく写真・映像で伝えるべきものであり、これがないとサイト全体の訴求力が著しく低下する。活動カードの「thumb」がすべて絵文字のグラデーションだけであるため、現在は実際の活動の様子が全くイメージできない。
- **修正方法**: `#members` セクションの後に `#gallery` セクションを追加。Instagram (@osakawind87) や Facebook から写真を取得し、CSS Grid または Masonry レイアウトでギャラリーを構築する。クリックで拡大するモーダル表示も検討する。

---

#### 3. 費用感の説明が著しく不十分

- **観点**: コンテンツの正確性、新入生向け訴求
- **詳細**: research.mdの「8. 制作メモ・注意事項」に「費用が高額なため、新入生へのコスト感の説明（月会費・道具代の目安）は丁寧に記載する」と明記されている。現状のFAQ回答（「詳しくは見学時にお気軽にご相談ください」）は費用の不透明さにより入部を躊躇わせる可能性がある。
- **修正方法**: FAQ の費用回答を改善し、「道具代はXX万円〜、遠征費は年間XX万円程度、大阪大学未来基金の支援対象」といった目安を記載する。正確な金額が不明な場合でも「初期費用・年間費用の内訳を見学時にご説明します」と案内するよりも「～万円台」の目安を示す方が信頼感につながる。

---

#### 4. `<meta description>` が設定されていない

- **観点**: SEO
- **詳細**: `<head>` 内に `<meta name="description">` タグが存在しない。これはGoogle等の検索結果に表示される説明文に直結し、SEO上・クリック率上の重大な欠落である。新歓シーズンに「大阪大学 ウインドサーフィン部」等で検索されることを考えると早急な対応が必要。
- **修正方法**: `<head>` に以下を追加する。
  ```html
  <meta name="description" content="大阪大学体育会ウインドサーフィン部の公式サイト。未経験者大歓迎、甲子園浜を拠点に週2回練習。インカレ入賞・総長賞受賞の本格派マリンスポーツクラブです。新入部員募集中。" />
  ```

---

#### 5. OGP（Open Graph Protocol）タグが設定されていない

- **観点**: SEO、新入生向け訴求
- **詳細**: Twitter/X・LINE・Slack等でURLをシェアした際にサムネイル・タイトル・説明が表示されない。新歓シーズンにSNSでシェアされることが多いため、OGPタグの欠落は拡散効果を著しく損なう。
- **修正方法**: `<head>` に以下を追加する。
  ```html
  <meta property="og:title" content="大阪大学体育会ウインドサーフィン部" />
  <meta property="og:description" content="未経験者大歓迎。甲子園浜から始まる最高のマリンライフ。新入部員募集中！" />
  <meta property="og:type" content="website" />
  <meta property="og:url" content="https://（サイトのURL）" />
  <meta property="og:image" content="https://（OGP画像のURL）" />
  <meta name="twitter:card" content="summary_large_image" />
  ```

#### 6. 地図（Googleマップ）がプレースホルダーのまま

- **観点**: デザイン・UX、情報の欠落
- **詳細**: `#access` セクションの地図エリアが「兵庫県西宮市 甲子園浜」と書かれた絵文字プレースホルダーになっており、実際の地図が埋め込まれていない。練習場所への行き方がわからない新入生にとって、場所の視覚的な確認は重要である。
- **修正方法**: Google Maps Embed API またはiframe埋め込みで甲子園浜の実際の地図を表示する。
  ```html
  <iframe
    src="https://www.google.com/maps/embed?pb=（甲子園浜の埋め込みURL）"
    width="100%" height="320" style="border:0; border-radius:16px;"
    allowfullscreen loading="lazy"
    referrerpolicy="no-referrer-when-downgrade"
    title="甲子園浜 練習場所マップ">
  </iframe>
  ```

#### 7. 活動カードの画像がすべて絵文字のみ

- **観点**: デザイン・UX、新入生向け訴求
- **詳細**: `#activities` セクションの6枚のカードサムネイル（`.activity-card-thumb`）は、いずれもグラデーション背景に単一の絵文字を置いただけの実装になっている。実際の練習・大会・遠征の写真に差し替えることで訴求力が大幅に向上する。
- **修正方法**: Instagram・Facebook から写真を入手し、`<img>` タグで各カードに設定する。写真が用意できない場合は、Unsplashなど著作権フリーのウインドサーフィン写真を一時的に使用する（商用利用注意）。

---

#### 8. FAQ の `faq-q` がボタン要素でなくdivであり、キーボード操作に非対応

- **観点**: アクセシビリティ
- **詳細**: FAQの質問行（`.faq-q`）は `<div>` 要素に `cursor: pointer` を設定してクリックイベントを付与しているが、セマンティックにはキーボードフォーカス・Enterキー操作ができないため、キーボードのみで操作するユーザーがアクセスできない。
- **修正方法**: `.faq-q` を `<div>` から `<button>` 要素に変更し、`type="button"` と `aria-expanded` 属性を付与する。
  ```html
  <button class="faq-q" type="button" aria-expanded="false">
    経験なしでも本当に大丈夫？<div class="arrow">&#9660;</div>
  </button>
  ```
  JavaScriptで `aria-expanded` を `true`/`false` に切り替える処理も追加する。

#### 9. `<nav>` のモバイルメニューに `role` と `aria-hidden` が設定されていない

- **観点**: アクセシビリティ
- **詳細**: `#mobileMenu` は表示/非表示が切り替わるが、非表示時に `aria-hidden="true"` が設定されず、スクリーンリーダーが非表示メニューのリンクを読み上げてしまう可能性がある。
- **修正方法**: JavaScriptのハンバーガートグル処理に以下を追加する。
  ```javascript
  mobileMenu.setAttribute('aria-hidden', !isOpen);
  ```
  初期状態では `aria-hidden="true"` をHTML側にも追加する。

#### 10. `<section id="hero">` のセマンティクスが不適切

- **観点**: HTML/CSS品質、アクセシビリティ
- **詳細**: ヒーローセクションが `<section>` タグで実装されているが、Heroセクションはページのメインコンテンツを示す `<main>` または `role="banner"` を持つ要素で包むべきである。また、ページ全体に `<main>` タグが存在しないため、スクリーンリーダーがメインコンテンツの開始点を把握できない。
- **修正方法**: `<nav>` の後ろから `</footer>` の前までを `<main>` タグで包む。
  ```html
  <main>
    <section id="hero">...</section>
    ...
  </main>
  ```

#### 11. `countUp` アニメーション関数がstatsセクションのunitタグを上書きする

- **観点**: HTML/CSS品質（JavaScript バグ）
- **詳細**: `countUp` 関数は `el.textContent = Math.floor(start) + suffix` で要素全体のテキストを上書きするため、`.stat-num` 内の `<span class="unit">` タグが消えてしまい、フォントサイズの差異（`0.5em`）による視覚的な単位表示が崩れる。実際には数字と単位がすべて同じサイズで表示されてしまう。
- **修正方法**: `countUp` 関数を `.stat-num` の数値部分（テキストノード）のみを更新するよう修正するか、HTMLを `<span class="num-value">14</span><span class="unit">年</span>` の構造に分離して、数値部分のみをアニメーションさせる。

#### 12. `<html lang="ja">` は設定済みだが `<title>` が短くSEO不十分

- **観点**: SEO
- **詳細**: `<title>` が「大阪大学体育会ウインドサーフィン部」のみであり、「新入部員募集」「ウインドサーフィン」「甲子園浜」などのキーワードが含まれていない。新歓シーズンに関連キーワードで検索ヒットする機会を逃している。
- **修正方法**: タイトルを以下のように変更する。
  ```html
  <title>大阪大学体育会ウインドサーフィン部 | 新入部員募集中 | 甲子園浜でウインドサーフィン</title>
  ```

#### 13. `<link rel="canonical">` が設定されていない

- **観点**: SEO
- **詳細**: 正規URLを示す `<link rel="canonical">` が `<head>` に設定されていない。URLの揺れ（www有無・末尾スラッシュ等）による重複コンテンツをGoogleが誤認する可能性がある。
- **修正方法**: `<head>` に以下を追加する。
  ```html
  <link rel="canonical" href="https://（正式なサイトのURL）" />
  ```

#### 14. フッターに大阪大学体育会へのリンクがない

- **観点**: 情報の欠落
- **詳細**: research.mdのサイト構成案のFOOTERに「大阪大学体育会リンク」が明記されているが、フッターの「Links」セクションに大阪大学体育会（https://www.osaka-u.ac.jp/ja/academics/eu_club/）へのリンクが存在しない。体育会公認クラブであることの信頼感を担保するリンクとして重要。
- **修正方法**: フッターの Links 欄に以下を追加する。
  ```html
  <a href="https://www.osaka-u.ac.jp/ja/academics/eu_club/" target="_blank" rel="noopener">大阪大学体育会</a>
  ```

### [重要度: 低]

#### 15. `<link rel="icon">` (favicon) が設定されていない

- **観点**: HTML/CSS品質、デザイン
- **詳細**: ブラウザタブに表示されるfaviconが設定されておらず、ブラウザのデフォルトアイコンが表示される。クラブサイトとしての完成度・ブランディングに影響する。
- **修正方法**: セイルや波モチーフのSVGアイコンを作成し、以下を `<head>` に追加する。
  ```html
  <link rel="icon" href="/favicon.svg" type="image/svg+xml" />
  <link rel="icon" href="/favicon.ico" />
  ```

---

#### 16. フォントのpreloadが設定されていない

- **観点**: HTML/CSS品質、パフォーマンス
- **詳細**: Google Fontsを `<link>` で読み込んでいるが、`rel="preload"` によるフォントの事前読み込みが設定されていない。特にモバイル回線では初回ロード時にFOUT（Flash of Unstyled Text）が発生する可能性がある。
- **修正方法**: `display=swap` はすでに設定されているため現状のFOUT対策は最低限あり。さらにパフォーマンスを高めたい場合、Google Fonts URL に `&display=optional` または自前ホスティングを検討する。

---

#### 17. モバイル表示でstatsセクションが2列になるが4項目のレイアウトが窮屈

- **観点**: モバイル対応
- **詳細**: `@media (max-width: 900px)` で `stats-grid` が `repeat(2, 1fr)` になるが、900px未満のタブレット幅で数字が大きいため（`clamp(2.5rem, 5vw, 4rem)`）横並びが窮屈になりうる。580px以下では特に問題ない。
- **修正方法**: 580px以下でも `repeat(2, 1fr)` のままか、必要なら `1fr` 一列にする。現状は `@media (max-width: 580px)` でstatsの列変更がないため、意図的であれば問題ない。念のため表示確認を推奨。

---

#### 18. ヒーローのCTAボタンに絵文字を使用しているがフォールバックがない

- **観点**: アクセシビリティ
- **詳細**: `&#128293;`（炎の絵文字）や `&#9654;`（再生ボタン）をボタンテキストに使用しているが、絵文字はスクリーンリーダーによって「fire」「black right pointing triangle」等と読み上げられ、意図が伝わらない場合がある。
- **修正方法**: 絵文字を `<span aria-hidden="true">` で包んで読み上げ対象から除外する。
  ```html
  <a href="#join" class="btn btn-primary">
    <span aria-hidden="true">🔥</span> 新入部員募集中
  </a>
  ```

---

#### 19. `<section id="stats">` に見出し要素がない

- **観点**: HTML/CSS品質、アクセシビリティ
- **詳細**: `#stats` セクションには `<h2>` などの見出しが存在せず、スクリーンリーダーユーザーがページアウトラインを把握しにくい。また、statsセクションは `<section>` として適切なaccesible nameを持つべきである。
- **修正方法**: visually-hiddenの見出しを追加するか、`aria-label` を `<section>` に付与する。
  ```html
  <section id="stats" aria-label="クラブの主な数値">
  ```

---

#### 20. research.mdで推奨されたInstagramフィード埋め込みが未実装

- **観点**: 情報の欠落、新入生向け訴求
- **詳細**: research.mdの「UIコンポーネントのアイディア」にInstagramフィード埋め込みが提案されているが未実装。新歓シーズン中のリアルタイム更新を伝える手段として効果的である。
- **修正方法**: Instagramの公式Embed機能（またはサードパーティウィジェット）を使用してギャラリーセクション内や新歓カード付近に埋め込みを追加する。ただし表示速度に影響するため、遅延読み込みを適用すること。

## 良い点（現状で評価できる実装）

- research.mdのカラーパレット（#0077B6 / #00B4D8 / #F77F00 / #1A1A2E / #F0F8FF）が忠実に実装されている
- Bebas Neue + Montserrat + Noto Sans JP というフォント構成がresearch.mdの推奨通りに実装されている
- ハンバーガーメニューのモバイル対応、`aria-label="メニュー"` の設定
- IntersectionObserver を使ったスクロールアニメーション（`fade-up`）の適切な実装
- FAQアコーディオンの実装（ただしアクセシビリティ改善は必要）
- `target="_blank"` に `rel="noopener"` を一貫して設定しておりセキュリティ面で適切
- countUpアニメーションによる数字の視覚的な演出
- ヒーローセクションのキャッチコピー「風を感じろ。速さを知れ。」がresearch.mdの候補の中から適切に選択されている
- join セクションの新入生向けメッセージがresearch.mdの紹介文をほぼ忠実に再現している
- 大阪大学未来基金へのリンクがFAQ・フッターの両方に記載されており費用支援の存在を示している
